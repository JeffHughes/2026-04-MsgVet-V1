6. Data Partitioning Strategy

Core principle: Every query is scoped by tenantId. No cross-tenant data access. Cosmos DB partition key = tenantId for all containers.

8. Key Invariants

  1. Tenant isolation: Every data access includes tenantId filter. No exceptions.
  2. Vet before send: No outbound message bypasses the vet pipeline (except emergency bypass with full audit).
  3. Kid inbound filtered: Every inbound message to a kid account runs the full filtering pipeline.
  4. Audit everything: Every state change produces an audit event with hash chain integrity.
  5. Idempotent sends: Every send operation uses an idempotency key. Duplicate calls return existing result.
  6. Deterministic first: Rule-based filters run before any LLM call to minimize cost and latency.
  7. Secrets never in DB: All credentials stored in Key Vault; DB stores only Key Vault references.
  8. Kid data minimal: COPPA-compliant: content storage off by default, auto-purge after retention window.

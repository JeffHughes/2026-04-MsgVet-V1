1. Problem Statement

For Kids

  • Children receive harmful content (bullying, predatory, explicit) with no real-time guardrails.
  • Parents lack visibility into messaging without invasive surveillance.
  • No platform offers both inbound filtering AND outbound safety in one product.

For Adults

  • Regret-sends cost relationships, jobs, and reputations daily.
  • No tool prevents messages to specific contacts during high-risk states.
  • Public posts lack pre-send review for tone, compliance, or appropriateness.

For Organizations

  • Executive communications require delegation with audit trails.
  • Compliance teams need message vetting without reading every message.
  • No platform combines governance + messaging + AI vetting in one system.

The Cost of Doing Nothing

ScenarioImpact
Kid exposed to predatory messagingCRITICAL Immeasurable harm
Drunk text to exHIGH Relationship damage, possible legal issues
Executive sends unvetted public statementHIGH Stock price, brand, legal
Employee sends privileged info externallyCRITICAL Regulatory fines, litigation

2. Goals, Non-goals & Success Metrics

Goals

  1. Stop bad messages before they go out — tone, content, time, audience, compliance.
  2. Stop bad messages on the way in — quarantine + review + alerts (especially for kids).
  3. Unify communications across accounts/channels with consistent enforcement.
  4. Support approvals + delegation (AI, human, quorum) with complete audit trail.
  5. Offer first-party chat + voice to bypass external channel constraints and reduce cost.
  6. COPPA compliance by design — verifiable parental consent, data minimization.
  7. Cost-optimized AI — two-tier inference, deterministic rules first, LLM second.

Non-goals (Explicit)

  • Reading all iOS SMS/iMessage system-wide (not possible for third-party apps).
  • Covert monitoring without consent.
  • Deceptive auto-replies (assistant must self-identify).
  • Replacing Slack/Teams — MsgVet governs, not replaces.

3. Personas & User Journeys

Kid Account (Primary MVP Start)

Journey: Kid receives suspicious message
1. Message arrives from unknown sender → 2. Vet engine: bullying + unknown = HIGH risk → 3. Quarantined (kid never sees it) → 4. Parent gets push notification → 5. Parent reviews: release / block / report

Adult Consumer

Journey: Drunk Guard activation
1. User enables Drunk Guard → 2. Composes text to ex at 2 AM → 3. Engine: ex-contact + drunk-guard + late = HOLD → 4. "Message held until 10 AM" → 5. Morning review: sees vet report (angry, not urgent), deletes

Corporate / Executive

Journey: CEO delegate sends public statement
1. EA drafts on CEO's behalf → 2. Requires quorum: PR + Legal → 3. Both approve → 4. Auto-sends via scheduled slot → 5. Full audit trail logged

7. Kid-First: Family Safety System

Build Priority: This is Milestone A. Everything else depends on proving kid safety works. COPPA compliance is non-negotiable.

Inbound Filtering Pipeline

Message Received │ ├─ Step 1: Deterministic Filters │ ├─ Allow list (trusted contacts) → ALLOW │ ├─ Deny list (blocked senders) → HARD BLOCK │ ├─ Time window (e.g., bedtime) → QUARANTINE │ └─ Unknown sender rules → QUARANTINE or escalate │ ├─ Step 2: LLM Classification (if not resolved) │ ├─ Categories: bullying, sexual, self-harm, threats, scams, PII │ ├─ Severity: LOW / MEDIUM / HIGH / CRITICAL │ └─ Confidence score │ └─ Step 3: Decision ├─ ALLOW (trusted + clean) ├─ ALLOW + FLAG (minor concern, logged) ├─ QUARANTINE (HIGH/CRITICAL; parent must release) └─ HARD BLOCK + metadata (known threats)

Parent Notification Tiers

SeverityNotificationTiming
CRITICAL Self-harm, threatsImmediate push + SMSReal-time
HIGH Bullying, sexualPush notification< 1 minute
MEDIUM Mild language, unknown senderBatched digestHourly

Crisis Escalation: If self-harm or threat signals detected → quarantine + parent alert + optional crisis resources (988 Lifeline, Crisis Text Line).

Kid Outbound Safety

11. Policy Engine & Rule Matrix

Rule Matrix Inputs

DimensionExamples
Account/ChannelGmail, SMS, X, MsgVet Chat
Target/GroupSpecific contact, group (family, work, public)
Time WindowAfter 11PM, weekends, work hours
LocationGeofence (opt-in): driving, in meeting
User StateManual toggles: drunk, angry, quiet, court mode
DirectionInbound, outbound, both

Resolution Rules

  1. Most specific wins: account + contact + time beats account-only.
  2. Stricter wins on conflict: more restrictive action at same specificity.
  3. Explicit override: override: true can relax (requires admin).
  4. Audit on every resolution: log which rules matched, won, and why.

TierModelsUse CasesEst. Cost
CheapGPT-4o-mini, Gemini Flash, HaikuStandard classification, basic rewrites~$0.01–0.03 / 1K msgs
StrongGPT-4o, Gemini Pro, SonnetEdge cases, legal/PR, polished rewrites~$0.10–0.30 / 1K msgs

Cost Controls

StrategySavings
Deterministic rules first (regex, allow/deny lists)30–50% messages need no LLM
Prompt caching (system prompts + policies)50–90% on cached tokens
Batch processing for non-interactive jobs50% batch discount
Minimal context (summaries, not full history)60–80% token reduction
Rewrite-on-demand only~70% rewrite cost elimination
Model routing: cheap first, escalate on low confidence80% cheap / 20% strong split

24. Build Order & Milestones

Milestone A: Kid-First Foundation Weeks 1–8

Family tenants, parent/child accounts, COPPA consent, kid policy engine, inbound filtering pipeline, outbound safety, quarantine system, parent dashboard, first-party chat MVP, logging, pagination, billing scaffolding, push notifications.

Milestone B: Adult Consumer Layer Weeks 9–14

Adult accounts, policy packs (Drunk/Ex/Professional), regret controls, intent capture + rewrites, scheduling service, alias groups, VoIP voice, email adapter, alias phone numbers.

Milestone C: Corporate Layer Weeks 15–20

Org tenants, role hierarchy, delegation model, quorum approvals, audit exports/retention, corporate policy templates, admin console, SSO prep.

Milestone D: Hardening & Launch Weeks 21–24

Load testing (10K concurrent, 100K msgs/hr), security audit, pen test, COPPA review, cost optimization, app store submissions, documentation, beta program (500 families, 50 corporate).
